# Saga DSL Grammar

Each `.saga` file is a self-contained module. The language uses Markdown-style formatting so story files are easy to read and write.

//****************************************/
// Example: inn.saga
//****************************************/

# Main
  `ale_count ?= 0` // Initializes ale_count only if it is nil.
  > The inn is crowded. The innkeeper is cleaning a glass at the bar.
  + -> [Speak to the innkeeper](#Innkeeper)
  + -> [Join the patrons and try to overhear rumors](#Patrons)
  + -> [Head out](/town_square.saga#Main)

# Patrons
  > You join the patrons for drinks and order an ale for yourself.
  `ale_count++` // Assigns 1 if ale_count is nil.
  `x = rand(100)` // x is a scene-local binding for this visit to Patrons.
  - `x > 80 && !man_ran` *-> [Continue](#.Man)
  - `x <= 20 && !met_woman` -> [Continue](#.WomanInRedDress)
  - -> [Continue](#.NoOne)

## Man
  `man_ran := true`
  > You pick out a man in the crowd.
  > He notices you and runs out the front door.
  + -> [Give chase](/town_square.saga#ChaseMan)
  + -> [Leave it be](#Main)

## WomanInRedDress
  > A woman in a red dress notices you, and walks over.
  + `!met_woman` -> [Introduce yourself](#.InitialConvo)
  + -> [Tip your cap and carry on](#Main)

### InitialConvo
  `met_woman := true`
  > The woman introduces herself as Clair. She tells you to go to the bank, and tell Joe that she sent you.
  > The woman turns away, and walks upstairs.
  + -> [Continue](#Main)

## NoOne
  > None of the bar patrons catch your interest.
  + -> [Continue](#Main)

# Innkeeper
  > The man looks at you gruffly.
  + `man_ran` *-> [Mention the man](#.AskAboutMan)
  + -> [Gruff back](#Main)

## AskAboutMan
  > The innkeeper lowers his voice. "Best not chase trouble through town."
  + -> [Continue](#Main)

//****************************************/
// Lexical Rules
//****************************************/

Scene labels are identifiers in the form `[A-Za-z_][A-Za-z0-9_]*`.

Expressions occur inside backticks:

```saga
`expression_text`
```

Blank lines are ignored.

Leading indentation before statements is ignored by the compiler. Authors may use indentation for readability, but indentation has no semantic meaning in v0.

Statements are single-line in v0. Multiline passages, choices, and effects are not supported.

For readability and simpler parsing, choice transfer arrows must be followed by whitespace.

Passage and choice text are trimmed of leading and trailing whitespace.

Choice targets store display text in `[label]` and the destination in `(destination)`.

Comments start with `//` when `//` appears outside a backtick expression and is either:

- at the beginning of a line after indentation, or
- after whitespace in a line.

Everything from `//` to the end of the line is ignored. `//` inside a backtick expression is a compile error in v0.

//****************************************/
// Scene Headings and Scope
//****************************************/

Markdown headings declare scenes:

```saga
# Heading
## NestedHeading
### DeeplyNestedHeading
```

Markdown hashes indicate a nested scene scope. A heading's full scene path is the concatenation of the nearest preceding headings at lower depths plus its own label. A heading at depth `N` replaces any current headings at depth `N` or deeper.

Every heading declares a scene.

Example:

```saga
# Patrons
## WomanInRedDress
### InitialConvo
## NoOne
```

Declares these scenes:

```text
Patrons
Patrons.WomanInRedDress
Patrons.WomanInRedDress.InitialConvo
Patrons.NoOne
```

//****************************************/
// Statements
//****************************************/

Scene statements execute top-to-bottom.

Conditions are evaluated when their statement is reached, after previous effects in the same scene have executed.

## Passage

A greater-than sign `>` is used for passages shown on screen:

```saga
> Passage text
> `show_condition` Passage text
```

In a passage, an optional condition immediately after `>` controls whether the passage is displayed. If the condition is falsey, the passage is skipped and scene evaluation continues to the next statement.

## Text Interpolation

Displayed text can include JavaScript expressions inside `#{...}`:

```saga
`player_name ?= prompt("What is your name?") || "traveler"`
> Welcome, #{player_name}.
>> [Guide](/characters.saga#Guide) As I was telling #{player_name}, the princess is in another castle.
+ -> [Ask #{player_name}'s question](#.Question)
```

Interpolation is evaluated at render time in passages, dialogue text, choice text, and image alt text. Values are inserted as text, not HTML.

Backticks remain the syntax for Saga conditions and effects. `#{...}` is only interpreted inside displayed text. In v0, interpolation expressions are simple JavaScript snippets and should not contain `}`.

## Dialogue

A double greater-than sign `>>` adds character dialogue. The first line in a dialogue bubble names a speaker target. Consecutive `>>` lines without a speaker continue the same bubble. Any non-dialogue statement ends the bubble.

```saga
>> [Blue Scarf](/characters.saga#BlueScarf) Then you are already late.
>>
>> `trust_blue_scarf` You came back. Good.
```

Speaker targets must use labeled destinations and must resolve to scenes decorated with `@widget std:character`.

## Widget Decorator

A scene can be marked as a widget by placing `@widget` immediately before its heading:

```saga
@widget std:contacts
# Contacts
  > People you have met
  + `met_blue_scarf` -> [Blue Scarf](/characters.saga#BlueScarf)
```

Built-in widget renderers are:

- `std:inventory` — dock widget for inventory-style choice lists.
- `std:contacts` — dock widget for people/contact lists.
- `std:status` — dock widget for status-style displays.
- `std:item` — modal widget for item details.
- `std:character` — modal/profile widget for characters and dialogue speakers.

## Image

Markdown image syntax renders an image from the story bundle:

```saga
![Alt text](/assets/images/example.png)
```

In v0, image paths may be root-relative to the story bundle or relative to the generated HTML file. Root-relative image paths are validated at compile time. When previewing a generated HTML file locally, root-relative image paths are resolved against the source story bundle.

## Choice

A plus sign `+` adds an additive choice button the player can select:

```saga
+ -> [Choice text](#Target)
+ `show_condition` -> [Choice text](#Target)
+ -> `enable_condition` [Choice text](#Target)
+ `show_condition` -> `enable_condition` [Choice text](#Target)
+ *-> [One-shot choice text](#.Child)
+ -> [Cross-module choice](/module.saga#Scene)
```

A minus sign `-` adds a fallback choice button. Fallback choices are evaluated top-to-bottom, and only the first visible fallback choice renders:

```saga
- `x > 80 && !man_ran` *-> [Continue](#.Man)
- `x <= 20 && !met_woman` -> [Continue](#.WomanInRedDress)
- -> [Continue](#.NoOne)
```

The choice grammar is:

```text
("+" | "-") show_condition? arrow enable_condition? [button text](destination)
```

In a choice, an optional condition immediately after `+` or `-` controls whether the choice is visible.

In a choice, an optional condition immediately after the arrow controls whether the choice is enabled.

If the choice is not visible, it is not shown to the player. If the choice is visible but not enabled, it is shown as disabled and cannot be selected.


## Effect

A backtick expression on its own line is an effect statement:

```saga
`effect expression`
```

Effects execute when the scene is entered and evaluation reaches the effect line.

The special `end:` destination ends the story and displays the choice text as the ending message:

```saga
- -> [You were eaten by a grue.](end:)
```

`end(...)` effects are still accepted for older stories, but new stories should prefer `end:` choices.

//****************************************/
// Transfers and Targets
//****************************************/

`+` choices all render if visible.

`-` choices are fallback choices: they are evaluated top-to-bottom, and only the first visible fallback choice renders.

`->` is a normal transfer/action symbol for choices.

`*->` marks a choice action as consumable once per playthrough.

When a choice target resolves to a widget scene, Saga automatically runs that widget action instead of replacing the current main scene. Dock widgets are activated, while modal widgets open in a modal.

For choices, `*->` is consumed only when the player selects the choice and transfers. On future visits, the consumed choice is no longer visible.

Choice and dialogue speaker links use Markdown-style labels with Saga destinations:

```saga
[Go to scene](#Scene)
[Go to child](#.Child)
[Go to parent](#..)
[Go to sibling](#..Sibling)
[Go to another module](/module.saga#Scene)
[End the story](end:)
```

The square brackets contain user-facing text. For scene destinations, the destination after `#` is the scene target. For `end:`, the square-bracket text is displayed as the final ending message.

A target without `.` is resolved from the module root. For example, from `Patrons.WomanInRedDress`, `#Main` resolves to `Main`, not `Patrons.WomanInRedDress.Main`.

In a scene target, `#.` resolves to the current scene.

In a scene target, `.Child` expands to a child of the current scene. For example, from `Patrons.WomanInRedDress`, `#.InitialConvo` resolves to `Patrons.WomanInRedDress.InitialConvo`.

In a scene target, `#..` resolves to the parent scene. For example, from `Patrons.WomanInRedDress.InitialConvo`, `#..` resolves to `Patrons.WomanInRedDress`. Using `#..` from a top-level scene is a compile error.

In a scene target, `..Sibling` resolves to a sibling of the current scene. For example, from `Patrons.WomanInRedDress.InitialConvo`, `#..FollowUp` resolves to `Patrons.WomanInRedDress.FollowUp`. Using `#..Sibling` from a top-level scene is a compile error; use `#Sibling` instead.

If a destination omits a module path, the current module is used. If a destination includes a module path, the scene reference is resolved in that module.

A destination includes the root-relative module path before `#`:

```saga
[Button text](/path/to/module.saga#Scene)
```

In v0, module paths are root-relative and must begin with `/`.

//****************************************/
// Expressions and State
//****************************************/

Expression parsing is TODO. For v0 parser work, backtick contents may be treated as opaque expression text.

Identifier lookup inside expressions follows this order:

1. Scene-local bindings.
2. Persistent story state.
3. `nil` if neither exists.

Scene-local bindings shadow persistent story state with the same name.

## Persistent initialize-if-nil

```saga
name ?= expr
```

Initializes persistent story state if `name` is currently nil. If `name` already exists, this statement is a no-op.

## Persistent assignment

```saga
name := expr
```

Assigns persistent story state.

## Scene-local binding

```saga
name = expr
```

Binds or rebinds a temporary local for the current scene visit. Locals are visible to subsequent expressions in the same scene visit. Locals are discarded when leaving the scene.

## Increment and decrement

```saga
name++
name--
```

If `name` resolves to nil, `name++` assigns `1` and `name--` assigns `-1`.

If `name` resolves to a number, `name++` increments it and `name--` decrements it.

If `name` resolves to a non-number value, increment and decrement are runtime errors.

Mutation operators apply to the nearest resolved binding. If a local binding exists, the local is mutated. Otherwise, persistent story state is mutated.

//****************************************/
// Scene Validity
//****************************************/

Each scene should have at least one possible completion path:

- one or more choices with transfers, or
- a reachable choice targeting `end:`.

Any statement after an unconditional ending is unreachable and should be reported as a semantic error.

//****************************************/
// Grammar Summary
//****************************************/

```text
module             = line*

line               = indentation? (blank | comment | decorator | heading | statement)

decorator          = "@widget" space widget_renderer
widget_renderer    = "std:inventory" | "std:contacts" | "std:status" | "std:item" | "std:character"

heading            = heading_marker space scene_label
heading_marker     = "#" { "#" }

statement          = passage
                   | image
                   | dialogue
                   | choice
                   | effect

passage            = ">" space show_condition? text
image              = "![" alt_text "](" image_path ")"
dialogue           = ">>" space show_condition? speaker_target? text

choice             = ("+" | "-") space show_condition? arrow space enable_condition? link_target

comment            = "//" text

effect             = expression

show_condition     = expression space
enable_condition   = expression space

expression         = "`" expression_text "`"

arrow              = "->" | "*->"

link_target        = "[" label_text "]" "(" destination ")"
destination        = "#" scene_ref
                   | module_path "#" scene_ref
                   | "end:"
module_path        = "/" path_text

scene_ref          = scene_path
                   | self_scene_path
                   | child_scene_path
                   | parent_scene_path
                   | sibling_scene_path
scene_path         = scene_label { "." scene_label }
self_scene_path    = "."
child_scene_path   = "." scene_path
parent_scene_path  = ".."
sibling_scene_path = ".." scene_path

scene_label        = /[A-Za-z_][A-Za-z0-9_]*/
label_text         = text
path_text          = text
```
